### Nginx Rate Limiting

Add to your `nginx.conf`:

```nginx
http {
    # Define rate limit zones
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=100r/m;
    limit_req_zone $binary_remote_addr zone=general_limit:10m rate=200r/m;
    
    server {
        location /api/ {
            limit_req zone=api_limit burst=10 nodelay;
            proxy_pass http://localhost:3000;
        }
    }
}
```

## ğŸ› Troubleshooting

### Issue: Puppeteer fails in production
**Error**: `libatk-1.0.so.0: cannot open shared object file`

**Solution**: Install Chrome dependencies (see Production Deployment section above)

**Alternative**: Set custom Chrome path
```bash
# Install Chrome
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt install ./google-chrome-stable_current_amd64.deb

# Set env variable
PUPPETEER_EXECUTABLE_PATH=/usr/bin/google-chrome-stable
```

### Issue: Getting 401 on protected routes
**Cause**: No valid session or NEXTAUTH_SECRET mismatch

**Solution**: 
- Check if logged in
- Verify NEXTAUTH_SECRET is same across app and middleware
- Check cookie settings (HTTPS in production requires secure cookies)
- Ensure session hasn't expired

### Issue: Rate limit hit too quickly
**Cause**: Multiple users on same IP or limit too low

**Solution**: 
- Increase limit in `middleware.ts`: `Ratelimit.slidingWindow(100, "10 m")`
- Check if multiple users sharing same IP (corporate networks)
- Review Upstash Redis dashboard for analytics
- Consider per-user rate limiting instead of per-IP

### Issue: MongoDB connection failed
**Error**: `MongoServerError: Authentication failed`

**Solution**:
- Verify MONGODB_URI is correct (check username/password)
- Check network access in MongoDB Atlas
- Ensure IP whitelist includes your server IP (or use 0.0.0.0/0 for all)
- Test connection: `mongosh "mongodb+srv://..."`

### Issue: OTP emails not sending
**Cause**: Invalid Gmail credentials or 2FA not enabled

**Solution**:
- Use Gmail App Password (not regular password)
- Enable 2-Factor Authentication on Gmail
- Generate App Password: Google Account â†’ Security â†’ 2-Step Verification â†’ App passwords
- Update EMAIL_USER and EMAIL_PASS in .env

### Issue: OAuth not working
**Cause**: Callback URL mismatch

**Solution**:
- Verify redirect URIs in Google/GitHub console
- Check NEXTAUTH_URL matches your domain
- Production: Update callback URLs to production domain
- Ensure providers are enabled in `lib/authOptions.ts`
